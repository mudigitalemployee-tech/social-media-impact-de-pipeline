The pipeline architecture is determined through an automated multi-signal scoring engine rather than human configuration, eliminating subjective bias and ensuring architectural alignment with the actual quantitative characteristics of the source data. The scoring engine evaluates five dimensions simultaneously: data volume (record count and file size informing partitioning and parallelism strategy), source system type (CSV, database, REST API, streaming : each carrying different latency, schema stability, and ingestion reliability characteristics), schema composition (column count, type diversity, PII presence, and null distribution informing transformation complexity), quality baseline (duplicate rate and null severity informing whether a lightweight Simple ETL or a full medallion separation is warranted), and business purpose (keywords in the declared purpose mapped to known analytical patterns : time-series analysis implies different architectural needs than cross-sectional segmentation). The architecture with the highest composite score is selected, with confidence levels reflecting the signal strength of the winning pattern relative to alternatives. Medallion Layer Configuration

The medallion architecture implements a three-tier data contract model where each layer serves a distinct operational purpose and enforces explicit quality guarantees at its output boundary. This separation of concerns is the architectural foundation of pipeline resilience: when a transformation logic change is required, only the affected Silver and Gold layers need reprocessing : Bronze, as the immutable raw record of what was received from the source, remains untouched and serves as the permanent reprocessing baseline. Bronze implements the "data landing zone" contract : zero transformation, full fidelity, append-only semantics. Every record received from the source lands in Bronze unchanged, including malformed rows, duplicates, and records that will ultimately fail DQ checks. This preserves auditability and enables complete reprocessing from source without re-querying external systems. Silver implements the "trusted data" contract : deduplicated, typed, null-handled, PII-masked, and DQ-validated records that satisfy the quality threshold required for safe analytical consumption. Silver is the layer that data scientists and analysts query when they need clean, reliable data without aggregation assumptions. Gold implements the "business-ready" contract : pre-computed KPIs, group-level aggregations, and denormalized metrics optimized for BI tool performance. Gold eliminates the need for complex runtime aggregations in dashboards, reducing query latency and ensuring metric consistency across all consumer surfaces.
